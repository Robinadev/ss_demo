PROBLEM
═══════════════════════════════════════════════════════════════════════════════
  Error: spawn xdg-open ENOENT
  
  Cause: npm/Node packages trying to open browsers in sandboxed environments
         (Windows, macOS, Vercel, CI/CD, Docker)

ROOT CAUSE ANALYSIS
═══════════════════════════════════════════════════════════════════════════════
  
  ┌─ npm package tries to open browser
  │  ├─ Uses: xdg-open (Linux only)
  │  ├─ Fails on: Windows, macOS, Vercel
  │  └─ Result: ENOENT (command not found) error
  │
  └─ Solution: Prevent browser opening attempt in first place

MULTI-LAYER SOLUTION
═══════════════════════════════════════════════════════════════════════════════

  LAYER 1: npm Configuration (.npmrc)
  ─────────────────────────────────────
    Setting: browser=none
    Effect: npm skips browser opening
    Coverage: Global (all npm commands)
    Status: ✓ Active
    
    .npmrc:
      browser=none
      legacy-peer-deps=true
      
  LAYER 2: Development Environment (.env.development.local)
  ───────────────────────────────────────────────────────────
    Settings:
      BROWSER=none
      CI=true
      SKIP_ENV_VALIDATION=true
    Effect: Node.js automatically loads these variables
    Coverage: Development mode
    Status: ✓ Active
    
    Automatic loading:
      $ node app.js
      # Node.js loads .env.development.local automatically
      
  LAYER 3: Script Error Handling (scripts/setup-db.js)
  ──────────────────────────────────────────────────────
    Feature: Error filtering
    Catches: xdg-open errors specifically
    Response: Logs as info, continues execution
    Coverage: Setup/database operations
    Status: ✓ Active
    
    Code:
      if (!err.message.includes('xdg-open')) {
        throw error;  // Real error
      } else {
        // Ignored - expected error
      }
      
  LAYER 4: Startup Scripts (start-development.sh/.bat)
  ──────────────────────────────────────────────────────
    Features:
      - Set BROWSER=none explicitly
      - Validate environment
      - Interactive menu
      - Colored output
    Coverage: Manual startup
    Status: ✓ Available
    
    Platforms:
      ✓ Linux/macOS: start-development.sh
      ✓ Windows: start-development.bat
      
  LAYER 5: Backend Configuration (backend/package.json)
  ─────────────────────────────────────────────────────
    Updates:
      - Prisma: --skip-generate flags
      - Scripts: Better error isolation
      - Naming: Consistent naming (dev vs start:dev)
    Coverage: Backend operations
    Status: ✓ Updated
